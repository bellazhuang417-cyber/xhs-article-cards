# QA Checklist

## 必过项

### 内容检查

- [ ] 每页卡片能独立成立，单独看也完整
- [ ] 没有把原文整段复制进去（已提炼压缩）
- [ ] 页数合理（4-8 页，不超过 9 页）
- [ ] 版式有节奏感（不连续两页同类型）
- [ ] 封面有配图
- [ ] 有结尾页
- [ ] **签名栏是用户自己的名字（来自 `user-config.json`），不是写死的任何名字**
- [ ] **底部 `sig-sub` 是当前文章标题，不是固定文字**
- [ ] **所有产物（HTML + 封面图 + 卡片 PNG）在同一个 `xhs-<slug>/` 目录里，没有散落**

### 视觉检查

- [ ] 3:4 比例（1080x1440 或 2160x2880）
- [ ] 中文正常显示，没有乱码
- [ ] 图片正常显示，没有裂图
- [ ] 文字没有溢出页面边界
- [ ] 每页都有顶部标签和底部签名
- [ ] 手绘边框完整显示

### 配图检查

- [ ] 配图不是 SVG 硬画的人物（如果是，必须换成 ImageGen 生成或定稿图）
- [ ] 配图风格统一（都是手绘素描风）
- [ ] 配图比例正确（**1024x1024 正方形**，不是 3:4 长方形）
- [ ] 配图背景是纯白（不是灰底/米色/渐变）
- [ ] **配图没有灰色边框**（ImageGen 原始输出常带灰框，必须裁掉）
- [ ] **配图没有"图片由AI生成"水印条**（底部灰色条带必须裁掉）
- [ ] **配图符合该用户的 `ip-constraints.md`**（发型/脸型/体型等固定特征不能偏离）

## 失败信号

### 内容失败

| 问题 | 原因 | 修复 |
|------|------|------|
| 页数太多（>9页） | 拆得太碎 | 合并同类页面，压缩到 6-8 页 |
| 页数太少（<3页） | 没拆开 | 按认知锚点拆成 4-5 页 |
| 文字太密 | 原文照搬 | 提炼压缩，每段 ≤80 字 |
| 没有节奏 | 全是正文页 | 交替使用 body/points/quote |
| 没有封面配图 | 忘了加 | 封面必须有配图 |

### 视觉失败

| 问题 | 原因 | 修复 |
|------|------|------|
| 中文乱码 | HTML 编码不是 UTF-8 | 用 Python 写入确保 UTF-8 |
| 图片不显示 | 路径错误或中文文件名 | 用英文文件名，图片放 HTML 同目录 |
| 文字溢出 | 内容太多 | 减少文字或拆成多页 |
| 边框缺失 | SVG 没渲染 | 检查 sketch-border SVG |

### 配图失败

| 问题 | 原因 | 修复 |
|------|------|------|
| SVG 人物车祸 | 用 SVG path 硬画人物 | 换成 ImageGen 生成或用定稿图 |
| 配图风格不统一 | 不同 prompt 生成 | 用统一的 prompt 模板 |
| 配图比例不对 | 生成的是 16:9 或 3:4 | prompt 里强调 1024x1024 正方形，size 参数用 "1024x1024" |
| 背景不是纯白 | 生成了灰底/米色 | prompt 里强调 pure white background |
| **灰框/水印未裁** | 直接用了 ImageGen 原始输出 | 生图后用 PIL 裁掉四周灰边 + 底部水印条 |
| **IP 特征偏离** | 没读 ip-constraints.md，prompt 里没写硬约束 | 生图前读 ip-constraints.md，用三重锁死法写入 prompt |

## 迭代方法

- **文字太密**：减少每页段落数，或拆成多页
- **配图车祸**：不要自己画，用 ImageGen 或定稿图
- **中文乱码**：用 Python `pathlib.Path.write_text(html, encoding='utf-8')` 写入
- **图片不显示**：图片放 HTML 同目录，用英文文件名
- **风格不统一**：所有配图用同一个 prompt 模板，只替换主题词
- **页数不对**：参考 split-strategy.md 的页数控制表

## 交付判断

高质量的卡片集应该让读者：
1. 看到封面想点进来
2. 顺滑地滑完所有页
3. 每页都有信息量，不水
4. 看完想收藏或关注

如果有一页让人觉得"这页没必要"，就删掉它。
