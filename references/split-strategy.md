# 拆页策略

## 核心原则

把文章拆成卡片，不是把文章切碎贴进去。每一页卡片要能独立成立——读者单独看这一页也能获得完整的信息点。

## 拆页步骤

### Step 1: 通读全文，标记认知锚点

读一遍文章，标记出：
- **核心判断**：作者最想表达的观点
- **认知转折**：但、然而、其实、真正的问题在于
- **方法论**：第一、第二、如何做
- **对比结构**：之前 vs 之后、错误 vs 正确
- **金句**：值得单独记住的话
- **总结**：文章的收束

### Step 2: 分配页面类型

把每个锚点分配到页面类型：

| 锚点类型 | 页面类型 | 说明 |
|---------|---------|------|
| 文章标题 + 核心观点 | cover | 封面 |
| 章节标题 | chapter | 章节过渡（可选） |
| 长文论述 | body | 正文页 |
| 编号要点/方法论 | points | 要点卡片 |
| 引用/金句 | quote | 金句页 |
| 总结收束 | ending | 结尾 |

### Step 3: 内容压缩

每页卡片的空间有限，必须压缩内容：

- **封面**：标题 ≤12 字，副标题 ≤20 字
- **正文页**：每段 ≤80 字，每页 2-3 段
- **要点页**：每个要点标题 ≤10 字，描述 ≤40 字
- **金句页**：金句 ≤30 字
- **结尾页**：总结 ≤60 字

**不要把原文整段复制进去。** 要提炼、压缩、改写成适合卡片阅读的短文本。

### Step 4: 排序与节奏

卡片的阅读节奏很重要：
- P1 封面：吸引注意
- P2 痛点/背景：建立共鸣
- P3-P(N-1) 核心内容：递进展开
- P(N) 结尾：收束 + 签名

避免连续两页都是同一种版式。交替使用 body / points / quote 来制造节奏感。

## 拆页示例

### 示例 1：工具分享类文章

```
原文结构：
- 我最近用了 Claude Code 的 Live Artifacts
- 痛点：数据分散，查找繁琐
- 核心能力 1：灵活组织
- 核心能力 2：AI 总结看点
- 团队应用 1：周报汇总
- 团队应用 2：自动催交通知
- 评论联动通知
- 总结：Claude Code 成了完整工作台

拆页方案（6 页）：
P1 cover    - 封面：Live Artifacts 运营数据工作台
P2 body     - 痛点：数据分散，查找繁琐
P3 points   - 核心能力：灵活组织 + AI 总结看点
P4 points   - 团队应用：周报汇总 + 自动催交
P5 body+points - 评论联动通知 + 闭环协作流
P6 ending   - 结尾：Claude Code 完整工作台
```

### 示例 2：读书笔记类文章

```
原文结构：
- 读了《XX》这本书
- 核心观点：阅读是认知的复利
- 阅读不是信息获取
- 阅读的真正价值
- 金句：我们读书不是为了记住，而是为了被改变
- 总结：阅读是漫长的复利游戏

拆页方案（5 页）：
P1 cover    - 封面：阅读是认知的复利
P2 chapter  - 第一章：阅读不是信息获取
P3 body     - 正文：阅读的真正价值
P4 quote    - 金句：我们读书不是为了记住
P5 ending   - 结尾：阅读是漫长的复利游戏
```

### 示例 3：方法论类文章

```
原文结构：
- 如何用 AI 做内容运营
- 方法 1：AI 辅助选题
- 方法 2：AI 辅助写作
- 方法 3：AI 辅助分发
- 常见误区
- 总结

拆页方案（6 页）：
P1 cover    - 封面：AI 内容运营方法论
P2 points   - 三大方法：选题 + 写作 + 分发
P3 body     - 方法 1 详解：AI 辅助选题
P4 body     - 方法 2 详解：AI 辅助写作
P5 body     - 方法 3 详解：AI 辅助分发
P6 ending   - 结尾：AI 是杠杆，不是替代
```

## 常见错误

1. **页数太多**：短文拆了 10 页，每页只有一两句话。→ 压缩到 4-5 页。
2. **页数太少**：长文只拆了 3 页，每页文字太密。→ 拆成 6-8 页。
3. **没有节奏**：连续 4 页都是正文页。→ 交替使用不同版式。
4. **照搬原文**：把原文整段复制进卡片。→ 提炼压缩成短文本。
5. **封面无聊**：封面只有标题没有插画。→ 封面必须有配图。
