# 配图指南

## 核心原则

配图是卡片的视觉锚点，不是装饰。好的配图让读者停下来看，坏的配图让读者划走。

## 配图优先级

```
① 已有 IP 定稿图（最佳——质量有保证，风格统一）
② ImageGen 生成（次选——根据文章主题定制）
③ 纯文字排版不加图（兜底——宁可不加，不要加车祸图）
```

**绝对不要用 SVG path 硬画人物/复杂插画。** 没有视觉感知力，画出来必然比例失调。

## 哪些页需要配图

| 页面类型 | 配图 | 说明 |
|---------|------|------|
| 封面 | ✅ 必须 | 封面没有图 = 没有视觉吸引力 |
| 章节页 | ⬜ 可选 | 一般不需要，大章节号本身就是视觉 |
| 正文页 | ⬜ 一般不需要 | 文字为主，加图会挤 |
| 要点页 | ❌ 不需要 | 要点卡片本身就是视觉结构 |
| 金句页 | ❌ 不需要 | 引用框是视觉主体 |
| 结尾页 | ⬜ 可选 | 如果封面有图，结尾一般不再加 |

## IP 使用规则

### 已有 IP 定稿图

如果用户有 IP 定稿图（见 `ip-constraints.md` 和 `user-config.json`），优先使用：

- **封面**：用和文章主题最相关的定稿图
  - 阅读/认知类 → 看书图
  - 生活/咖啡类 → 喝咖啡图
  - 旅行/探索类 → 旅行图
- 定稿图放在封面左侧 420px 区域
- 用 `<img src="定稿图名.png">` 引用，不要用 SVG 重画

### 没有定稿图时（需要为当前文章生成新动作/场景的 IP 图）

用 ImageGen 生成。生成时：
- **比例：1024x1024 正方形**（不是 3:4 长方形——长方形会撑破封面 420px 区域）
- 极简手绘风格（thin ink pen sketch）
- 纯白背景，大量留白
- 人物占画面 40-60%
- 线条有自然抖动感
- 不要过于精致/完美线条
- 不要灰底、渐变、填充色块

### ⚠️ IP 特征约束（每次生图必读）

**每次生图前，必须先读取 `ip-constraints.md`（skill 根目录）。** 这个文件记录了该用户 IP 的固定特征，是所有生图的"宪法"。

生成 IP 形象时（无论是封面还是其他页面），以下规则**必须遵守**：

1. **读取约束**：生图前先读 `ip-constraints.md`，把里面的固定特征（发型、脸型、眼睛、体型、风格）逐条提取出来
2. **写入 prompt**：把特征转成英文写入 prompt，用三重锁死法：
   - ✅ 正描述（如：short bob hair ending at jawline）
   - ❌ 否定式（如：NOT long hair, NOT shoulder-length）
   - 🤔 兜底（如：when unsure, make it SHORTER rather than longer）
   - 正参照（如：Japanese bob haircut）
3. **不硬编码任何用户的具体特征**：skill 本身不带任何具体发型/脸型——这些都从 `ip-constraints.md` 动态读取。每个用户的 IP 约束不同。

**为什么：** IP 的固定特征（尤其发型）是模型最容易"自由发挥"出错的地方。光在 prompt 里写特征词不够——必须用否定式 + 正参照 + "拿不准就往保守方向调"三重锁死。把这些约束固化成文件后，无论哪个模型、哪次生图，都能读到同一套标准。

**比例（硬约束）：**
- 用 `size: "1024x1024"`（正方形），不用 1024x1536（3:4）
- 原因：封面 `.cover-illu` 区域是固定 420px 宽的正方形容器，3:4 图会被压扁或留白

**生图后处理（必须执行）：**
ImageGen 的输出常带两层多余内容：
1. **四周灰色边框**（约 150-170px 宽，灰度约 RGB(203,203,201)）
2. **底部"图片由AI生成"水印条**（约 100-150px 高的灰色条带）

生图后必须用 Python PIL 自动裁剪，只保留人物+装饰元素：
```python
from PIL import Image
img = Image.open('生成图.png')
# 扫描亮度 > 235 的区域为白底内容区
# 裁掉四周灰框 + 底部水印条
# 保存覆盖原文件
```
不裁剪直接用 → 封面会出现"三层灰框"问题（模板手绘边框 + 图片灰边 + 水印条）。

## 风格 DNA（复刻自 ian-xiaohei，适配 3:4）

### 必须

- 3:4 竖版配图（适配小红书卡片）
- 纯白背景：不要米色、暖灰、纸张纹理、渐变
- 黑色手绘线稿为主：细线、轻微抖动、不机械
- 大量留白：主体占画面约 40%-60%
- 一张图只讲一个核心场景
- 人物在做某个动作（看书/喝咖啡/工作/旅行），不是静态站姿

### 颜色

- 黑色：主体线稿、角色、结构
- 橙色：重点标注、路径、箭头
- 蓝色：补充说明、第二层信息
- 颜色要克制，宁可少不要多

### 绝对不要

- 不要商业插画
- 不要 PPT 信息图
- 不要可爱卡通
- 不要复杂背景、渐变、阴影
- 不要 SVG 硬画人物（能力边界，画出来必然车祸）
- 不要过于精致的矢量风格

## 构图模式

### 场景图（封面用）

适合：封面配图。

画法：一个人物在做和文章主题相关的动作。
- 阅读 → 看书
- 咖啡 → 喝咖啡
- 工作 → 在电脑前
- 旅行 → 背包走路
- 思考 → 托腮看远方

### 隐喻图（可选）

适合：文章核心概念的可视化。

画法：把抽象概念换成物理动作。
- 卡住、漏掉、分拣、沉淀、发酵、开门、折叠、拆包
- 用低科技物件：纸箱、抽屉、旧机器、漏斗、秤、门、梯子

### 对比图（可选）

适合：前后对比、错误 vs 正确。

画法：左混乱，右有序，中间箭头。

## 原创隐喻生成法

每次都从当前文章重新发明隐喻，不能照搬旧图。

### 三步

1. 把抽象概念换成一个物理动作：卡住、漏掉、变重、分拣、沉淀、发酵、开门、折叠、拆包、回流。
2. 把系统结构换成一个低科技物件：坏掉的机器、纸箱、抽屉、水管、邮筒、怪表盘、秤、井、梯子。
3. 让人物承担动作：不是站旁边，而是卡在机器里、拉错线、守门、搬运、修补。

## 反复刻规则

不要直接复用已有案例的构图。每次都要从当前文章重新发明一个视觉隐喻。

同类主题也要换新隐喻。例如"知识积累"不一定画书堆，可以画一个人在往瓶子里倒东西；"效率提升"不一定画齿轮，可以画一个人在整理抽屉。
